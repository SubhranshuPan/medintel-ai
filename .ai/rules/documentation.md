# Documentation Rules

## Purpose

Documentation is treated as a first-class project artifact.

---

# Source of Truth

Documentation hierarchy:

1. /docs
2. /.ai/llm-wiki
3. Source Code

If conflicts exist, `/docs` takes precedence.

---

# Documentation Principles

- Keep documentation concise.
- Avoid duplication.
- Update documentation alongside implementation.
- Link related documents instead of repeating content.

---

# AI Workspace

The `.ai` workspace contains AI-optimized summaries.

It must never replace or contradict the documentation in `/docs`.

---

# When to Update Documentation

Update documentation when:

- Adding a feature.
- Changing architecture.
- Modifying APIs.
- Changing repository structure.
- Making significant engineering decisions.

---

# Quality Standards

Documentation should be:

- Accurate
- Current
- Actionable
- Easy to navigate
- Consistent
