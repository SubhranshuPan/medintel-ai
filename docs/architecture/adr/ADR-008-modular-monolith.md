# ADR-008 — Modular Monolith

## Status

Accepted

## Context

The project requires a modular monolith architecture.

## Decision

Modular monolith was chosen as the architecture.

## Alternatives Considered

- Microservices
- Layered Monolith

## Consequences

### Positive

- Excellent async support
- Automatic OpenAPI generation
- Strong typing
- High performance

### Negative

- Smaller ecosystem than Django
- Requires familiarity with async programming
